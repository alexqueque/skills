#### **Use Cases**

The various user classes identified the following primary actors and use cases for the COS:

| Primary actor   | Use cases                                                                                                                                                                                  |
|-----------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Patron          | 1. Order a Meal<br>2. Change Meal Order<br>3. Cancel Meal Order<br>4. View Menu<br>5. Register for Payroll Deduction<br>6. Unregister for Payroll Deduction<br>7. Manage Meal Subscription |
| Menu Manager    | 8. Create a Menu<br>9. Modify a Menu<br>10. Delete a Menu<br>11. Archive Menus<br>12. Define a Meal Special                                                                                |
| Cafeteria Staff | 13. Prepare Meal<br>14. Generate a Payment Request<br>15. Request Meal Delivery<br>16. Generate System Usage Reports                                                                       |
| Meal Deliverer  | 17. Record Meal Delivery<br>18. Print Delivery Instructions                                                                                                                                |

| Primary Actor:  | Patron                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 | Secondary Actors: | Cafeteria Inventory System |  |
| Description:    | A Patron accesses the Cafeteria Ordering System from either the corporate intranet or<br>external Internet, views the menu for a specific date, selects food items, and places an<br>order for a meal to be picked up in the cafeteria or delivered to a specified location within a<br>specified 15-minute time window.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |                   |                            |  |
| Trigger:        | A Patron indicates that he wants to order a meal.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |                   |                            |  |
| Preconditions:  | PRE-1. Patron is logged into COS.<br>PRE-2. Patron is registered for meal payments by payroll deduction.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |                   |                            |  |
| Postconditions: | POST-1. Meal order is stored in COS with a status of "Accepted."<br>POST-2. Inventory of available food items is updated to reflect items in this order.<br>POST-3. Remaining delivery capacity for the requested time window is updated.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |                   |                            |  |
| Normal Flow:    | 1.0 Order a Single Meal<br>1. Patron asks to view menu for a specific date. (see 1.0.E1, 1.0.E2)<br>2. COS displays menu of available food items and the daily special.<br>3. Patron selects one or more food items from menu. (see 1.1)<br>4. Patron indicates that meal order is complete. (see 1.2)<br>5. COS displays ordered menu items, individual prices, and total price, including taxes and<br>delivery charge.<br>6. Patron either confirms meal order (continue normal flow) or requests to modify meal order<br>(return to step 2).<br>7. COS displays available delivery times for the delivery date.<br>8. Patron selects a delivery time and specifies the delivery location.<br>9. Patron specifies payment method.<br>10. COS confirms acceptance of the order.<br>11. COS sends Patron an email message confirming order details, price, and delivery<br>instructions.<br>12. COS stores order, sends food item information to Cafeteria Inventory System, and updates<br>available delivery times. |                   |                            |  |

| Alternative Flows: | 1.1 Order multiple identical meals<br>1. Patron requests a specified number of identical meals. (see 1.1.E1)<br>2. Return to step 4 of normal flow.<br>1.2 Order multiple meals<br>1. Patron asks to order another meal.<br>2. Return to step 1 of normal flow.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |  |
|--------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--|
| Exceptions:        | 1.0.E1 Requested date is today and current time is after today's order cutoff time<br>1. COS informs Patron that it's too late to place an order for today.<br>2a. If Patron cancels the meal ordering process, then COS terminates use case.<br>2b. Else if Patron requests another date, then COS restarts use case.<br>1.0.E2 No delivery times left<br>1. COS informs Patron that no delivery times are available for the meal date.<br>2a. If Patron cancels the meal ordering process, then COS terminates use case.<br>2b. Else if Patron requests to pick the order up at the cafeteria, then continue with normal<br>flow, but skip steps 7 and 8.<br>1.1.E1 Insufficient inventory to fulfill multiple meal order<br>1. COS informs Patron of the maximum number of identical meals he can order, based on<br>current available inventory.<br>2a. If Patron modifies number of meals ordered, then return to step 4 of normal flow.<br>2b. Else if Patron cancels the meal ordering process, then COS terminates use case. |  |
| Priority:          | High                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |  |

| Other Information: | 1. Patron shall be able to cancel the meal ordering process at any time prior to confirming it.<br>2. Patron shall be able to view all meals he ordered within the previous six months and repeat<br>one of those meals as the new order, provided that all food items are available on the<br>menu for the requested delivery date. (Priority = medium) [Note: You could also show this<br>as an alternative flow for the use case.]<br>3. The default date is the current date if the Patron is using the system before today's order<br>cutoff time. Otherwise, the default date is the next day that the cafeteria is open.                                                                                                                                                                                                                                                                                                                                                                                                      |  |
| Assumptions:       | Assume that 15 percent of Patrons will order the daily special (Source: previous 6 months of<br>cafeteria data).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |  |
