# HEX BOSC SRS Generator

## Overview

生成符合上海银行（BOSC）V4.8 标准的高质量业务需求书（SRS）。本 skill 旨在帮助需求分析人员和业务经理创建结构清晰、内容完整且符合银行标准的业务需求文档。

## Core Workflow

当用户请求创建 BOSC 业务需求书时，请遵循以下流程：

### Step 1: Gather Context (获取背景)

在生成文档前，收集以下核心信息：
- **需求背景**：业务现状、流程断点、痛点。
- **需求目标**：可量化的后评估指标（线上化、数字化、智能化）。
- **功能清单**：拆分为最小业务单位。
- **用户信息与渠道**：用户数及发起渠道。

### Step 2: Generate SRS Structure (生成结构)

使用 `references/srs_template.md` 模板创建文档。必须包含以下核心章节：
1. **需求概述** - 背景、目标（含后评估指标）、总体需求描述、条目清单。
2. **功能详述** - 业务功能需求模板（用户信息、流程图、控制逻辑、后端处理）、界面展示、要素清单、流程埋点。
3. **关联影响** - 会计核算、数据支撑、监管风险、运营支撑等分析。

### Step 3: Detailed Functional Design (详细功能设计)

对于每个功能条目，采用 BOSC 规范格式：
- **控制逻辑**：明确流程环节和业务规则。
- **输入输出要素**：参考《附件0：输入输出要素模板》，列出字段名、输入方式、是否必输等。
- **后端处理**：强调数据流向、敏感字段脱敏及安全审计要求。

### Step 4: Cross-Impact Analysis (关联影响分析)

填写关联影响分析表，详细描述：
- **会计核算**：核算需求、内部账号、中收等。
- **监管风险**：监管及风险要求、征信/监管报送等。
- **运营支撑**：业务运营监控、集中运营诉求等。

### Step 5: Validate & Review (验证与评审)

运行验证脚本确保文档符合 V4.8 模板要求：
```bash
scripts/validate_srs.sh <srs_file.md>
```
