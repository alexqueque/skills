# Hex Vision Scope

## 概述

生成全面、结构严谨的“愿景与范围 (Vision and Scope)”文档。本技能通过定义业务需求、利益相关者、产品愿景和详细的项目范围，帮助项目经理和业务分析师为项目的成功奠定基础。

## 核心工作流

当用户请求创建愿景与范围文档时（例如：“为我们的新移动支付系统创建愿景与范围文档”），请遵循以下工作流：

### 第一步：收集背景信息

在生成文档之前，通过发现对话收集基本信息。如果用户已提供详细简报，可根据需要调整提问。

**关键提问列表：**
1. **当前背景**：目前的情况是什么？面临的主要痛点有哪些？
2. **业务机会**：我们要解决什么问题？它的核心价值是什么？
3. **业务目标**：我们希望实现什么具体的、可衡量的结果？
4. **成功标准**：如何衡量项目是否成功？
5. **愿景陈述**：谁是目标用户？产品解决了什么问题？与现有方案有何不同？
6. **主要功能**：产品将具备哪些核心功能？
7. **发布计划**：初步计划分几个阶段发布？
8. **限制与排除**：有哪些功能是明确不做的？

### 第二步：生成文档结构

使用 `references/vision_scope_template.md` 中的模板创建文档。文档应包括：

1. **业务需求 (Business Requirements)**：背景、机会、业务目标、成功指标、愿景陈述、业务风险。
2. **范围与限制 (Scope and Limitations)**：主要功能、版本发布规划、局限性与排除项。
3. **业务上下文 (Business Context)**：利益相关者分析、项目优先级、部署注意事项。

### 第三步：定义业务目标 (Planguage)

对于关键业务目标，尽可能使用 **Planguage** 进行精确描述：
- **Scale (规模)**：衡量指标是什么？
- **Meter (测量仪)**：如何获取数据？
- **Past (过去值)**：当前的基准值。
- **Goal (目标值)**：我们希望达到的目标。

### 第四步：进行验证与审核

完成后，用户可以运行验证脚本检查文档的完整性：

```bash
skills/hex-vision-scope/scripts/validate_vision_scope.sh <文件名.md>
```

## 使用场景示例

### 场景 1：新项目启动
**用户请求**：“我要做一个面向餐饮业的在线订餐系统，帮我写愿景与范围文档。”
**执行**：
1. 询问背景、痛点（如：排队时间长、浪费严重）。
2. 使用模板生成初稿。
3. 参考 `references/Vision-Scop-example.md` 提供类似餐饮系统的示例。

### 场景 2：功能扩展
**用户请求**：“我们需要在现有电商系统中增加跨境支付功能，请更新范围文档。”
**执行**：
1. 确认新增功能的业务机会。
2. 更新版本发布规划表（Release Plan）。
3. 明确指出排除项（例如：暂不支持加密货币支付）。

## 愿景与范围编写最佳实践

### 编写质量要求
- **愿景要清晰**：一句话说明产品做什么、为谁做。
- **目标要量化**：避免使用“提高效率”等模糊词，使用“效率提升 20%”。
- **范围要明确**：列出“排除项”与列出“功能项”同样重要，以防止范围蔓延。
- **利益相关者要全面**：考虑不仅是最终用户，还有运营、财务、法律等干系人。
