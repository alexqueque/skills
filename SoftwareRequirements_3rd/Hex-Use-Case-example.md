| Normal Flow:    | 1.0 Order a Single Meal<br>1. Patron asks to view menu for a specific date. (see 1.0.E1, 1.0.E2)<br>2. COS displays menu of available food items and the daily special.<br>3. COS displays ordered menu items, individual prices, and total price, including taxes and<br>delivery charge.<br>6. Patron either confirms meal order (continue normal flow) or requests to modify meal order<br>(return to step 2).<br>7. COS displays available delivery times for the delivery date.<br>8. Patron selects a delivery time and specifies the delivery location.<br>9. Patron specifies payment method.<br>10. COS confirms acceptance of the order.<br>11. COS sends Patron an email message confirming order details, price, and delivery<br>instructions.<br>12. COS stores order, sends food item information to Cafeteria Inventory System, and updates<br>available delivery times. |                   |                            |  |

| Alternative Flows: | 1.1 Order multiple identical meals<br>1. Patron requests a specified number of identical meals. (see 1.1.E1)<br>2. |--------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--|

COS informs Patron that no delivery times are available for the meal date.<br>2a. If Patron cancels the meal ordering process, then COS terminates use case.<br>2b. Patron is registered for payroll deduction.                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |                   |                    |

Payroll System confirms that Patron is eligible to register for payroll deduction.<br>3. COS asks Patron to confirm his desire to register for payroll deduction.<br>4. If so, COS asks Payroll System to establish payroll deduction for Patron.<br>5. Payroll System confirms that payroll deduction is established.<br>6.
